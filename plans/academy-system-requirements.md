# ข้อกำหนดและฟีเจอร์สำหรับโมดูลโรงเรียน (Academy)

เพื่อให้เห็นภาพที่ชัดเจนก่อนเริ่มสร้าง ตามแนวคิดที่ว่า **"โรงเรียนคือกลุ่มใหญ่กลุ่มหนึ่ง"** นี่คือข้อเสนอรายละเอียดของฟีเจอร์และกฎเกณฑ์ต่างๆ ในภาษาไทยครับ

## 1. ใครสามารถเปิดโรงเรียน (Academy) ได้บ้าง?

เพื่อให้ระบบมีความน่าเชื่อถือและจัดการได้ง่าย ผมขอเสนอทางเลือกดังนี้:

- **ทางเลือก A (เสรี)**: สมาชิกทุกคนสามารถเปิดโรงเรียนได้ (อาจต้องใช้แต้มสะสมจำนวนหนึ่งในการเปิด)
- **ทางเลือก B (ตรวจสอบ)**: ผู้ที่ได้รับการยืนยันว่าเป็น "ครู/ผู้สอน" เท่านั้นที่สามารถเปิดโรงเรียนหลักได้
- **ข้อสรุป**: ในระดับเทคนิค เราจะอนุญาตให้ผู้สร้าง (Owner) กำหนดได้เองว่าใครจะเป็นผู้บริหาร (Admins) ต่อไป

## 2. ฟีเจอร์หลักที่ "หนึ่งโรงเรียน" ต้องมี

### ก. ศูนย์รวมข่าวสาร (School Feed)

- **Announcements**: โพสต์ประกาศจากโรงเรียน (เฉพาะ Admin/ครู ที่โพสต์ได้หรือตั้งค่าให้สมาชิกโพสต์ได้)
- **Interaction**: การกด Like, Love, Happy (ตามที่เราปรับ UI ไป) เพื่อสร้างการมีส่วนร่วม

### ข. ฝ่ายงานและห้องเรียน (Departments & Sub-Groups)

- **ฝ่ายงาน (Departments)**: รองรับการสร้างกลุ่มย่อยแยกตามฝ่าย เช่น ฝ่ายวิชาการ, ฝ่ายบริหาร, ฝ่ายการเงิน
- **ห้องเรียน (Classrooms)**: สำหรับจัดการนักเรียนตามระดับชั้น
- **ฟังก์ชัน**: แต่ละฝ่าย/ห้องจะมีพื้นที่เฉพาะสำหรับจัดการเอกสารและสื่อสารกันเอง

### ค. การจัดการหลักสูตร (Curriculum Management)

- **การสร้างหลักสูตร**: กำหนดชื่อหลักสูตร และรายวิชาที่ต้องเรียนภายใต้หลักสูตรนั้นๆ
- **ความเชื่อมโยง**: สนับสนุนการจัดกลุ่มวิชา (Courses) เข้าสู่หลักสูตรที่เป็นระบบ

### ง. กิจกรรมและการเช็คชื่อ (Activities & Attendance)

- **การสร้างกิจกรรม**: ระบุวัน เวลา และสถานที่สำหรับกิจกรรมของโรงเรียน
- **การเช็คชื่อ (Attendance)**: ระบบบันทึกการเข้าร่วมกิจกรรมของนักเรียน เพื่อนำไปประเมินผล
- **การแจ้งเตือน**: แจ้งเตือนนักเรียนเมื่อมีกิจกรรมใหม่ที่ต้องเข้าร่วม

### จ. ระบบบริการและสวัสดิการ (Future Services)

เพื่อรองรับความต้องการที่หลากหลายในอนาคต ระบบจะออกแบบให้ขยายตัวสู่ส่วนงานเหล่านี้:

- **ระบบสหกรณ์โรงเรียน (School Cooperative)**: จัดการการออม การกู้ยืม และปันผลสำหรับสมาชิก (ใช้ PP/Gold เป็นฐาน)
- **ระบบร้านค้าโรงเรียน (School Shop)**: ระบบขายสินค้าและพัสดุทางการศึกษา พร้อมระบบสต็อกสินค้าเบื้องต้น
- **ระบบหอพัก (Dormitory Management)**: จัดการการเข้าพัก การเช็คชื่อในหอพัก และข้อมูลที่พักของนักเรียน

### ฉ. คลังวิชา (Course Library)

- แสดงรายการคอร์สเรียนทั้งหมดที่สังกัดโรงเรียนนี้ หรือตามหลักสูตรที่เลือก
- จัดหมวดหมู่ตามรายวิชาหรือระดับชั้น

### ช. ระบบจัดการสมาชิก (Member Management)

- **Student Profiles**: เชื่อมโยงกับข้อมูลวิชาการ ผลการเรียน ข้อมูลหอพัก และคะแนน PP/Gold
- **Staff Directory**: รายชื่อครูและเจ้าหน้าที่ พร้อมบทบาทหน้าที่แยกตามฝ่ายงาน (วิชาการ, การเงิน, บริการ)

## 3. ข้อกำหนดในการเข้าโรงเรียน (Entry Rules)

Admin ของโรงเรียนสามารถเลือกรูปแบบการเข้าถึงได้ 4 รูปแบบ:

1. **สาธารณะ (Public)**: ใครก็เข้าได้ทันที
2. **ต้องขออนุมัติ (Approval Required)**: ผู้ใช้กดขอเข้าร่วม แล้ว Admin ตรวจสอบก่อนอนุมัติ
3. **ใช้แต้มแลกเข้า (Point Entry)**: ต้องใช้แต้ม PP/Gold ตามที่โรงเรียนกำหนดในการสมัครเข้า
4. **ส่วนตัว/เชิญเท่านั้น (Private/Invite Only)**: ไม่แสดงในหน้าค้นหา ต้องใช้ลิงก์เชิญเท่านั้น

## 4. บทบาทภายในโรงเรียน (Roles Hierarchy)

เพื่อให้สอดคล้องกับโครงสร้างกลุ่ม เราแบ่งระดับดังนี้:

- **เจ้าของ (Owner/Principal)**: จัดการได้ทุกอย่าง รวมถึงการตั้งค่าโรงเรียนและการเงิน
- **ผู้ดูแล (Admin/Vice-Principal)**: ช่วยจัดการสมาชิก อนุมัติการเข้าเรียน และจัดการหน้าฟีด
- **ผู้สอน (Teacher)**: จัดการคอร์สเรียน โพสต์บทเรียน และตรวจงานในห้องเรียนที่ได้รับมอบหมาย
- **นักเรียน (Student)**: เข้าถึงเนื้อหา ส่งงาน และมีส่วนร่วมในกิจกรรม

---

> [!NOTE]
> ด้วยสถาปัตยกรรมแบบกลุ่มรวม (Universal Group) เราจะสามารถย้อนคืน (Rollback) หรือโอนย้ายสมาชิกจากกลุ่มหนึ่งไปอีกกลุ่มหนึ่งได้ง่าย เพราะใช้มาตรฐานข้อมูลเดียวกันครับ
